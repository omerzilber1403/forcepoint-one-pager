## Design Constraints

- **One scrolling page.** No client-side routing to sub-pages.
- **Dark theme only.** Accent: electric indigo (`#6366f1`) + cyan (`#22d3ee`).
- **No placeholder lorem ipsum** in the production build.
- **Mobile-first.** Breakpoints: 375px / 768px / 1280px.
- **Lighthouse targets:** Performance 90+, Accessibility 95+, SEO 100.

---

## Component Sourcing Rule

All UI primitives must come from **21st.dev community components** or **Aceternity UI** (the primary library surfaced via 21st.dev).

**No homemade bare-bones components** unless zero alternatives exist. If you are about to write a basic card or button from scratch, stop and find a 21st.dev/Aceternity equivalent first.

Every component used must be logged in `ux-ui/component-registry.md`.

---

## Data Privacy Rule

- No real PII used in any demo or test dataset
- No real confidential company names or proprietary data
- All demo data is synthetic
- The Data Privacy section on the portfolio must exist and speak directly to Forcepoint's use cases
